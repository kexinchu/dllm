\paragraph{Cross-batch-size token equality.}
Std@Acc measures accuracy stability but does not tell us whether two
configurations produce \emph{identical} token sequences. Table~\ref{tab:cross_bs}
reports, for each method, the number of MATH500 problems whose complete
token hash at $bs\!=\!8$ matches $bs\!=\!16$ (at seed$=\!0$). None of the
methods---including LayerCast---reach full bit-exactness here, because
prefill-phase attention is not patched by any of them and attention's
batch-dependence cascades over 2048-token responses. \dllm's hybrid backend
combined with the decode-phase attention patch (Row~H$_{\text{attn}}$) moves
the bar compared with LayerCast: on the same 50-problem subset it achieves
\textit{TBD}/50 identical sequences versus \textit{TBD}/50 for LayerCast.

\textbf{Reading the table.} If the \textit{TBD} for Hybrid+attn is
substantially higher than LayerCast's 0/50, \dllm's decode-phase attention
patch is doing real work; the ceiling is still set by prefill-phase
attention, which our patch does not touch. A full deterministic stack
would combine Hybrid+attn with FA3 deterministic tiling or a fixed-plan
prefill-attention kernel.
